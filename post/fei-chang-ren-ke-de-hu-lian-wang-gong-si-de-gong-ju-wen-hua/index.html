非常认可的互联网公司的工具文化

2018-02-28 7 min read # 职场

有关公司的生产力工具文化的文章，原文：那些豌豆荚教会我用的工作利器。

1. 邮件使用公约

简要原则

  1. 把事情说清楚
  2. 简短，但短不要为了而短
  3. 邮件标题、内容易于检索
  4. 注重阅读体验
  5. 多说事实，少说道理
  6. 发出去的邮件，泼出去的水，发邮件前三思

关于写邮件

写邮件的最佳顺序应该是：附件、标题、正文、收件人，以免遗漏，最后写收件人亦可避免还没写完就不小心发出去了。记得检查错别字和链接。

非常重要和紧急的事情在标题中加【重要】【紧急】，火烧眉毛的事直接当面说、打电话。

内部邮件无需客套（无指向性：Hi all；个人：Hi xxx)。

在邮件开头说清楚事情 — 倒金字塔的叙事：用一段话，言简意赅地概括大意，讲清楚「四个 W 和一个 H」— 谁（Who）、什么（What）、何时（When）、为什么（Why）、怎么样（How）。

复杂的事情分成几个大块，每块下面几个小点，以重要性和紧急程度排序，把不可遗漏的重点用黄底标出。

交代清楚事情的 Deadline，不使用「越快越好」

减去、增加收件人时，用「-xxx + xxx」。

2. 开会指南

会议组织者：

  • **邀请正确的人。**人越少越好，如果你不知道该邀请谁，请提前搞清楚，而不是邀请一个 group 的人来参与。
  • 尽量提前发出会议邀请（Google calendar）。会议邀请中需要明确会议的 topic 和 agenda。可选的会议参与者，请在会议邀请上标注为 optional。
  • **准时开始。**并且指定好会议记录人员。会议开始前简单介绍会议的 agenda。
  • 控制好时间，准时结束。需要控制会议的讨论时间，不要过于发散而导致会议时间延长，也避免会议室预定时间到后被别人赶出来。
  • 确保会后 1 个小时内将会议记录发送给所有与会者。如果有后续的 Action，也需要在会议记录中指定 Owner 和 Deadline。
  • 多个会议室部署了双屏幕，其中一个屏幕可以用来显示会议笔记。会议笔记记录人可以把笔记实时投影在其中一个屏幕上，这样子其他与会者能实时进行补充和修订，能很大提升工作效率。

会议参与者：

  • 在收到会议邀请后，如果你不是 optional 的，意味着你必须参加。
  • 请不要参加你不知道是去干什么的会议。如果你自己认为不需要参加，请和组织者沟通，达成共识（而不是选择到时候不去）。
  • 如果你是 mandatory 的，请准时参加会议。如果不能准时参加，请请假并说明原因。
  • 请不要带电脑（除非和会议相关的主题需要用到），请把手机调到震动。

3. 文案指南

在产品设计方面，也有一些大家可以遵循的规则在指导着我们日常的工作，比如我们有自己的文案委员会来为产品文案把关，也有成型的文档可以供参考。

这是文书表述上的一些文字格式规范，目的在于增强文字的可读性和豌豆荚产品及豌豆们对外形象的一致性，适用范围包括但不限于软件界面、帮助文档等严肃内容平台，博客和微博等网络宣传媒体平台的用词风格可较轻松活泼些。本指南会不断增补修订。

标点符号

  1. 中文文书原则上请使用全角标点符号，英文文书请使用半角标点符号。中英夹杂原则上请使用全角标点符号。下面有另行规定者例外。

  2. 括号的使用：中文文书中，括号中包括数字时采用半角括号，并在括号外留一个空格。例如：「全部联系人（12) J。

  3. 英文文书逗号和句号后请留一个半角空格；中英夹杂文书中英文（包括数字）与汉字之间留一个半角空格。

    • a. 正确：「这是 1 只 HTC 手机」错误：「这是1只HTC手机」
    • b. 正确：「2011 年 8 月 14 日」错误：「2011年8月14日」

  4. 百分号写法：56.8%是对的，56.8 %是错的。百分号和它前面的数字之间不留空格。

  5. 段落之间使用 <p> 标签分隔（一般来说实施效果是，段落间的视觉空隙为一倍 line-height)，段首不要空格。

  6. 不要使用「。。。」代替「……」，也不要使用三个英文句点「.」代替 「…」。

  7. 引号一律使用「」，而不是 “”。

  8. 严禁出现「！！」，尽量避免使用「！」。请先冷静下来再坐电脑前敲键盘。

  9. 长数字的书写：从小数点起，向左和向右每三位数字一组，组间空格。示例：2 748 456。

4. Design Hackathon 和 OKR

还有包含头脑风暴环节的 Design Hackathon，作为产品定义阶段和团队所有人一起就目标进行广泛发散和沟通的工具，以及在目标制定和总结阶段使用的 OKR 工具。

头脑风暴其实是最难的一部分。

常见的头脑风暴都是大家在一起，你一言我一语的讨论。但是发散思维和群体讨论都是两件很容易失控的事情，所以一般的头脑风暴容易存在以下问题：

  1. 辩论会：对别人的想法进行判断，觉得「不靠谱」，甚至变成了辩论会
  2. 一言堂：强势的参与者，通常是老板，说「这个好」，然后大家都顺着这个思路往下做
  3. 假大空：提了一些类似「一定要做好产品」这样的想法
  4. 各说各的：大家都有自己的思路，不从别人的想法上展开，失去了群体讨论的意义
  5. 自我限制：「这样实现成本太高」、「这样可能别的部门不支持」

所以在豌豆英做头脑风暴的时候，我们有个规则是：不！要！讨！论！

所有人提出的 idea 需要写在 post 贴上，最好是画出来。采用 N x 5 x 5 的方式。N 表示所有参与头脑风暴的人总数，这个式子表示需要每人在 5 min 的时间内写下 5 个想法，然后将这 5 个想法传给下一个人，同时接收上一个人传来的 post 贴，再写一轮，如此类推，N 轮过后，每个人手中都会有 5N 个想法，所有人共有 5 x N x N 个。所有人都要写，但是相互之间不要交流，这样子其实避免了常见的几个问题：

  1. 没有讨论，辩论会的情况就不会发生。
  2. 没有讨论，老板想要完全控制方向是不太可能的。
  3. 传递 post 贴，所有人都会看到之前其他人的想法并且一次为基础发散，站在巨人的肩膀上了。
  4. 五分钟内五个想法，是个高强度的脑力训练，强迫参与者打破一切限制，寻找任何可能的思路，参与者往往会比较积极的参考别人的想法。

上面的这些工具帮助我们在产品的定义、执行、总结各个阶段更加高效、默契地配合，降低了摩擦和误解的概率，提高了整体的运作效率。

越来越多的公司开始意识到：工具是公司文化价值观的一部分，大家也逐渐学会用好的工具去传达「尊重员工」、「凡事都有不同的做法」和「让员工也感受到爱」等观念。

我同意文中的观点，使用好工具是为了提高生产效率，让工作更加轻松有趣。
